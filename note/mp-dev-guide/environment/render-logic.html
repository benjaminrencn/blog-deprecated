# 通信模型

  • 渲染层、逻辑层由2个线程管理
  • 渲染层
    • 由 WebView 渲染
    • 多个界面存在多个 WebView 线程
  • 逻辑层
    • 由 JsScore 线程运行
  • 两个线程的通信经由 Native（微信客户端）中转
  • 逻辑层发送网络请求 Native 转发

# 数据驱动

  • 绑定状态和视图
  • WXML 结构转 JS 对象再转 DOM 树
  • setData 改变数据，产生的 JS 对象对应节点发生变化，比较差异并应用到 DOM 树上已更新视图

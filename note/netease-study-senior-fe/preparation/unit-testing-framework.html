# 单元测试框架选型

测试框架 + 断言库 + mock 库 -> 编写测试用例

# 测试框架

  • jasmine

    • BDD 风格
    • 功能全面，自带 assert、mock 功能

  • jest

    • 基于 jasmine
    • 对于 react 友好

  • mocha

    • TJ 出品
    • 可以运行在 node 端、浏览器端
    • 需要自己驾培断言库等

  • Qunit

    • 出自 jquery，后独立不依赖 jquery
    • 已被其他框架超越

  • intern

    • SitePen 出品
    • 功能齐全
    • 新兴小众

  • jasmine、mocha 比较

    • jasmine
      • 开箱即用，自带断言、mock 等功能
      • 不易扩展
    • mocha
      • 灵活可扩展
      • 集成度低需要自己配置

# 断言库

  • chai
    • 流行
    • 支持 BDD、TDD，全面
  • should
    • TJ 出品
    • API 语义好
    • 文档不足
  • expect
    • should 子集
    • 没亮点
  • assert
    • node 核心模块，node 环境可以使用
    • 语法弱

# mock 库

  • sinon
    • 功能强大

  • 语法

    • describe
      • 对应测试的模块/函数
      • 参数1 描述
      • 参数2 执行方法
    • it
      • 对应测试用例（test case）
      • 参数1 描述
      • 参数2 执行方法
    describe('approve when amount < 5', () => {
  it('should return right statement when amount < 5', () => {
    expect(approve({ amount: 3 })).to.eql('主任审批通过！');
  });
});

# chai

  • 安装

    yarn add -D chai
    1

  • 语法

    • BDD

      • expect
      expect(approve({ amount: 3 })).to.eql('主任审批通过');
      1
      • should
      approve({ amount: 3 }).should.eql('主任生审批通过');
      1

    • TDD

      • assert
      assert.equal(approve({ amount: 3 }), '主任生审批通过');
      1

  • 语法

    • Spies
      • 提供函数调用信息，但不改变函数行为
    let ajaxSpy = sinon.spy(jQuery, 'ajax');

async.getUser((data) => {
  data.user.name.should.equal('小明');
});
console.log(ajaxSpy.callCount);

ajaxSpy.restore();

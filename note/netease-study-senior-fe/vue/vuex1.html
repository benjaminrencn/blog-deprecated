# Vuex

# 背景

  • Vue 2.0 是单向数据流的
  • 父组件 - prop -> 子组件
  • 父组件 <- $emit - 子组件
  • 兄弟组件需要通过父组件传递数据
    复杂场景下开发效率过高
  • $parent、$root 耦合程度高，不利于复用

# 概念

  • 使用单一状态树
  • State
    • 唯一数据源 store 实例
    • 与 vue 实例 data 遵循相同规则
    • 响应式的
    • vue 实例通过 computed 返回 Vuex 的值
  • Getter
    • 类似 store 的 computed 计算属性
    • 根据依赖缓存，依赖值发生该变才会重新计算
  • Mutation
    • 更改 Vuex store 中的状态唯一方法
    • 由 commit 触发
    • 同步操作
  • Action
    • 由 dispatch 触发
    • 异步操作
    • 通过 Mutation 改变状态

# API

# Vuex.Store 实例方法

  • commit
    提交 mutation
  • dispatch 分发 action

# 组件绑定的辅助函数

  • mapState
    • 创建计算属性以返回 Vuex store 中状态
  • mapGetters
    • 创建计算属性以返回 getter 返回值
  • mapMutations
    • 创建组件方法提交 mutation
  • mapActions
    • 创建组件方法分发 action

import Vue from 'vue'
import Vuex from 'vuex'

Vue.use(Vuex)

export default new Vuex.Store({
  state: {
    count: 0,
  },
  getters: {
    doubleCount: (state) => state.count * 2
  },
  mutations: {
    addCount(state, payload = 1) {
      state.count += payload
    }
  },
  actions: {
    asyncAddCount({ commit }) {
      setTimeout(() => {
        commit('addCount')
      }, 1000)
    }
  },
  modules: {
  }
})
